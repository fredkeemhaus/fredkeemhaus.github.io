Python pipenv를 활용해보자 🍖

Pipenv을 사용하기 앞서

python을 사용하면서, pip, virtualenv를 사용하게 된다.

pip은 python을 이용하며 사용하게 되는 패키지 관리 툴이다. node js를 사용할 때, npm, yarn등과 같다고 생각하면 된다.

virtualenv는 python 프로젝트를 프로젝트별로 고립된 개발환경을 구축해주기 때문에 특정환경에 맞게 지웠다 만들었다 할 필요없이 개발할 수 있는 환경이 된다.

하지만, 또 생기는 문제점은 pip패키지들을 설치하는데 있어서 여간 귀찮지 않을 수 없다. 그 귀찮음을 덜어주기 위해, requirements.txt라는 파일을 통해 통으로 패키지를 관리할 수 있도록 지원한다고 하나, requirements.txt를 버전을 명시하지 않고 작성하였을 때, 똑같이 설치된다는 보장이 없고 일일이 버전을 기재하는 것 또한 귀찮음이 생긴다.

Pipenv의 등장

이름만 들어도 pip과 virtualenv가 합쳐진 듯한 느낌을 준다. 합쳐진 것이 맞다.

  • pip 그리고, virtualenv를 따로 쓸 필요가 없다.

  • Pipenv는 Pipfile과 Pipfile.lock을 requirements.txt를 대신하여 사용한다.

  • 해쉬가 자동으로 생성된다. (보안)

  • .env 파일들을 사용한 스트림라인 개발 워크플로우

  • pyenv가 사용 가능하다면, 필요한 pytho도 자동으로 설치.

  • Pipfile을 찾으면서 자동으로 프로젝트 홈을 찾아준다.

  • Pipfile이 없다면 자동으로 생성

  • 패키지를 설치/삭제하면, nodeJS의 package.json 경우와 마찬가지로 Pipfile에서 추가/삭제한다.

  • 자동으로 .env 파일을 인식.
